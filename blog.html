Git Topics: .git and filesystem issues

November 7, 2017

Recently at work, I ran into one of those once-in-a-blue-moon git issues that generally end up being solved with creative Googling and git magic. I wanted to understand what went wrong, so I took a deeper dive into git's internals and learned something useful.

              
                >> git pull origin master
                [nothing happens]

Git uses the filesystem to organize references to objects that it wants to track. What this means is that if you use filesystem-like slashes in a branch name, git will use that path to store the reference on your system. This isn't a problem unless you try to have branch names that aren't at consistent depths.
